# Cross-agency A2A — paper §6 stand-ins

The paper's §6.1 (emergency response) and §6.2 (supply chain) use
cases both describe agent-to-agent (A2A) collaboration that
crosses organisational boundaries — a county hazmat unit
consulting on a chemical signal, a supplier's negotiation agent
responding to a procurement request. The runtime is a
single-process reference implementation, so it does not actually
run two separate organisations' agents talking over a network.
Instead, it models the **mesh-level semantics** of cross-boundary
collaboration.

## What is modelled

- **The capability needed to talk out** is itself a tool the
  envelope governs. In `emergency_response.yaml`, the
  hazmat_coordinator does not start with
  `consult_external_agency` — it must acquire it via `request_grant`
  (R2), with mandatory HITL approval and a stated reason. The mesh
  refuses any outbound consultation that did not go through that
  grant.
- **The outbound message itself crosses a mesh boundary.** In
  `supply_chain.yaml`, `send_supplier_message` is marked
  `boundary: egress`, and `envelope.boundary_policies` inspect the
  message before the MCP tool sees it. Inventory, urgency, sole-source,
  and consumption leaks are blocked; pricing-strategy terms are
  sanitized. The deterministic `comms_guardrail` script agent remains
  as a preflight reformulation helper, but the mesh is the
  load-bearing enforcement point.
- **A platform-initiated checkpoint watches the cadence.** Four or
  more outbound messages in a single procurement cycle triggers a
  human review — the §6.2 drift cue.

## What is not modelled

There is no real network call to another agency or supplier. The
"external" endpoint records to a local outbound log; the
counter-offers that drive negotiation are seeded in the human-
supplied brief, not generated by a counter-party agent. Full A2A
would require a remote-node `kind` and a transport that preserves
the mesh chokepoint across the boundary — listed as a future kind
in `native_vs_script_agents.md`, not implemented here.

## Why this is still meaningful

Even without a real remote endpoint, the runtime demonstrates the
load-bearing claim: the *envelope* and the mesh's boundary policy —
not the agent's prompt — are what keep an outbound action from
happening without the right authority, the right approval, and the
right content review. Replace the local recorder with a real network
sender and the §6 sovereignty story holds.
